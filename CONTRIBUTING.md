# Contributing

Thanks for your interest in contributing to SecondBrain. This project is intentionally small and aims to be a friendly place for contributors.

How to contribute
- File an issue for bugs or feature requests.
- Fork the repo and open a pull request with a focused change.
- Keep changes atomic and include tests or manual verification steps where possible.

Code style
- C code: C99, avoid non-portable compiler extensions.
- Use clear function names and add comments for non-obvious behavior.

Building
- See docs/BUILD.md for desktop and Android build instructions.

License
- By contributing you agree to license your contributions under the project's MIT license.
